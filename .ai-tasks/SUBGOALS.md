# MG1 有序子目標

1. **盤點來源與資料契約**
   - 讀取 `hermes-agent-corp` 的 `/office` 頁面、直接依賴與靜態資產。
   - 記錄 REST 回應、WebSocket 事件格式及畫面所需欄位。
   - 產出最小移植範圍，排除 routing、其他頁面及全域基礎設施。
   - 依賴：無。

2. **建立最小前端腳手架**
   - 建立 Vite + React 18 + TypeScript 專案。
   - 加入 Tailwind CSS v3、基本腳本、`zh-Hant` HTML 設定與 `.gitignore`。
   - 確認初始 build 與 lint 通過。
   - 依賴：子目標 1，以免先加入不必要套件。

3. **定義後端型別與連線層**
   - 依實際契約建立 Office 資料型別。
   - 以原生 `fetch` 實作初始狀態讀取。
   - 以原生 `WebSocket` 實作事件接收、連線狀態、清理及最小重連。
   - 支援兩個 Vite 環境變數及 localhost 預設值。
   - 依賴：子目標 1、2。

4. **移植 Office 核心介面**
   - 移植必要版面、元件及資產。
   - 接上初始狀態與即時事件更新。
   - 補齊繁體中文的載入、空資料、錯誤及斷線狀態。
   - 不移植來源專案中與 Office 無關的抽象或依賴。
   - 依賴：子目標 3。

5. **完成品質與瀏覽器驗證**
   - 驗證桌面與窄螢幕版面、鍵盤操作及基本可存取性。
   - 執行 build、lint，以及資料解析存在非平凡分支時的一個最小自動化測試。
   - 連接實際後端驗證初始載入、即時更新、異常訊息、斷線與重連。
   - 依賴：子目標 4。

6. **整理 GitHub 交付**
   - 補充最小啟動說明、環境變數與後端前置條件。
   - 確認沒有秘密、建置產物、未使用依賴或超出範圍的檔案。
   - 準備推送至 `chung233/ai-office`。
   - 依賴：子目標 5。

執行順序為 `1 → 2 → 3 → 4 → 5 → 6`；沒有可安全並行且能省下實際工時的前置工作。
